What is React? ‚öõÔ∏è

React is a JavaScript library for building user interfaces, especially single-page applications. Created by Facebook, it's component-based and uses a virtual DOM for efficient updates.

Key Features:

  • Component-Based: Build encapsulated components
  • Virtual DOM: Efficient rendering and updates
  • Declarative: Describe what UI should look like
  • Learn Once, Write Anywhere: Web, mobile (React Native)
  • Strong Ecosystem: Huge community and libraries

Virtual DOM

React creates a virtual representation of DOM in memory, making updates fast

Components

Reusable pieces of UI that can manage their own state

JSX

JavaScript syntax extension that looks like HTML

One-Way Data Flow

Data flows down from parent to child components

üöÄ Why React?

  • Used by Facebook, Netflix, Airbnb, Uber
  • Huge job market demand
  • Excellent developer tools
  • Active community and regular updates
  • Great performance with virtual DOM

JSX Syntax üìù

JSX is a syntax extension for JavaScript that looks like HTML. It makes React components more readable and writable.

JSX Basics:

// JSX example
const element = <h1>Hello, React! ‚öõÔ∏è</h1>;

// JSX with variables
const name = 'Developer';
const greeting = <h1>Hello, {name}!</h1>;

// JSX with expressions
const user = {
    firstName: 'John',
    lastName: 'Doe'
};

const element2 = (
    <h1>
        Hello, {user.firstName} {user.lastName}!
    </h1>
);

// JSX with functions
function formatName(user) {
    return user.firstName + ' ' + user.lastName;
}

const element3 = <h1>Hello, {formatName(user)}!</h1>;

JSX Rules:

// 1. Must return single parent element
const validJSX = (
    <div>
        <h1>Title</h1>
        <p>Paragraph</p>
    </div>
);

// Or use React Fragment
const fragmentJSX = (
    <React.Fragment>
        <h1>Title</h1>
        <p>Paragraph</p>
    </React.Fragment>
);

// Short syntax for Fragment
const shortFragment = (
    <>
        <h1>Title</h1>
        <p>Paragraph</p>
    </>
);

// 2. Use className instead of class
const styledElement = <div className="container">Content</div>;

// 3. Use camelCase for attributes
const inputElement = (
    <input 
        type="text" 
        placeholder="Enter text"
        onChange={handleChange}
        autoComplete="off"
    />
);

// 4. Self-closing tags must have /
const imageElement = <img src="image.jpg" alt="Description" />;

Embedding JavaScript:

function App() {
    const isLoggedIn = true;
    const users = ['Alice', 'Bob', 'Charlie'];
    const currentTime = new Date().toLocaleTimeString();
    
    return (
        <div>
            {/* Comments in JSX */}
            <h1>Welcome to React!</h1>
            
            {/* Conditional rendering */}
            {isLoggedIn ? <p>Welcome back!</p> : <p>Please log in</p>}
            
            {/* Expressions */}
            <p>Current time: {currentTime}</p>
            <p>2 + 2 = {2 + 2}</p>
            
            {/* Arrays */}
            <p>User count: {users.length}</p>
            
            {/* Inline styles (object) */}
            <div style={{
                backgroundColor: 'lightblue',
                padding: '10px',
                borderRadius: '5px'
            }}>
                Styled with inline styles
            </div>
        </div>
    );
}

üí° JSX Best Practices:

  • Use parentheses for multi-line JSX
  • Use descriptive variable names
  • Keep JSX readable - break complex expressions into variables
  • Use React Developer Tools for debugging
  • Remember: JSX is JavaScript, not HTML!

React Components üß©

Components are the building blocks of React applications. They let you split the UI into independent, reusable pieces.

Function Components (Modern way):

// Simple function component
function Welcome() {
    return <h1>Hello, World!</h1>;
}

// Arrow function component
const Greeting = () => {
    return <h2>Welcome to React! ‚öõÔ∏è</h2>;
};

// Component with logic
function CurrentTime() {
    const now = new Date();
    const timeString = now.toLocaleTimeString();
    
    return (
        <div>
            <h3>Current Time</h3>
            <p>{timeString}</p>
        </div>
    );
}

// Using components
function App() {
    return (
        <div>
            <Welcome />
            <Greeting />
            <CurrentTime />
        </div>
    );
}

Class Components (Legacy, but still used):

import React, { Component } from 'react';

class Welcome extends Component {
    render() {
        return <h1>Hello from Class Component!</h1>;
    }
}

class Clock extends Component {
    constructor(props) {
        super(props);
        this.state = {
            time: new Date()
        };
    }
    
    componentDidMount() {
        this.timer = setInterval(() => {
            this.setState({ time: new Date() });
        }, 1000);
    }
    
    componentWillUnmount() {
        clearInterval(this.timer);
    }
    
    render() {
        return (
            <div>
                <h2>Current Time:</h2>
                <p>{this.state.time.toLocaleTimeString()}</p>
            </div>
        );
    }
}

Props (Properties) üì¶

Props are how you pass data from parent components to child components. They're read-only and help make components reusable.

Basic Props Usage:

// Child component receiving props
function UserCard({ name, email, age, avatar }) {
    return (
        <div className="user-card">
            <img src={avatar} alt={`${name}'s avatar`} />
            <h3>{name}</h3>
            <p>Email: {email}</p>
            <p>Age: {age}</p>
        </div>
    );
}

// Parent component passing props
function App() {

        <div>

                name="John Doe"

                age={30}

            <UserCard 
                name="Jane Smith"
                email="jane@example.com"
                age={25}
                avatar="https://example.com/jane.jpg"

State & useState Hook üéõÔ∏è

State allows components to change their data over time. The useState hook is the modern way to add state to function components.

Basic useState:

import React, { useState } from 'react';

function Counter() {
    // useState returns [currentValue, setterFunction]
    const [count, setCount] = useState(0);
    
    const increment = () => {
        setCount(count + 1);
    };
    
    const decrement = () => {
        setCount(count - 1);
    };
    
    const reset = () => {
        setCount(0);
    };
    
    return (
        <div>
            <h2>Counter: {count}</h2>
            <button onClick={increment}>+1</button>
            <button onClick={decrement}>-1</button>
            <button onClick={reset}>Reset</button>
        </div>
    );
}

Multiple State Variables:

function UserProfile() {
    const [name, setName] = useState('');
    const [email, setEmail] = useState('');
    const [age, setAge] = useState(0);
    const [isVisible, setIsVisible] = useState(true);
    
    const handleSubmit = (e) => {
        e.preventDefault();
        console.log({ name, email, age });
    };
    
    return (
        <div>
            {isVisible && (
                <form onSubmit={handleSubmit}>
                    <input
                        type="text"
                        placeholder="Name"
                        value={name}
                        onChange={(e) => setName(e.target.value)}
                    />
                    <input
                        type="email"
                        placeholder="Email"
                        value={email}
                        onChange={(e) => setEmail(e.target.value)}
                    />
                    <input
                        type="number"
                        placeholder="Age"
                        value={age}
                        onChange={(e) => setAge(parseInt(e.target.value) || 0)}
                    />
                    <button type="submit">Save Profile</button>
                </form>
            )}
            <button onClick={() => setIsVisible(!isVisible)}>
                {isVisible ? 'Hide' : 'Show'} Form
            </button>
        </div>
    );
}

Object State:

function UserForm() {
    const [user, setUser] = useState({
        name: '',
        email: '',
        preferences: {
            theme: 'light',
            notifications: true
        }
    });
    
    const handleInputChange = (field, value) => {
        setUser(prevUser => ({
            ...prevUser,
            [field]: value
        }));
    };
    
    const handlePreferenceChange = (field, value) => {
        setUser(prevUser => ({
            ...prevUser,
            preferences: {
                ...prevUser.preferences,
                [field]: value
            }
        }));
    };
    
    return (
        <div>
            <h2>User Settings</h2>
            <input
                type="text"
                placeholder="Name"
                value={user.name}
                onChange={(e) => handleInputChange('name', e.target.value)}
            />
            <input
                type="email"
                placeholder="Email"
                value={user.email}
                onChange={(e) => handleInputChange('email', e.target.value)}
            />
            
            <h3>Preferences</h3>
            <select
                value={user.preferences.theme}
                onChange={(e) => handlePreferenceChange('theme', e.target.value)}
            >
                <option value="light">Light</option>
                <option value="dark">Dark</option>
            </select>
            
            <label>
                <input
                    type="checkbox"
                    checked={user.preferences.notifications}
                    onChange={(e) => handlePreferenceChange('notifications', e.target.checked)}
                />
                Enable Notifications
            </label>
            
            <pre>{JSON.stringify(user, null, 2)}</pre>
        </div>
    );
}

üéõÔ∏è State Best Practices:

  • Use multiple useState calls for unrelated state
  • Always use the setter function to update state
  • Use functional updates when new state depends on previous state
  • Keep state as simple as possible
  • Don't mutate state objects directly - use spread operator
  • Consider useReducer for complex state logic
