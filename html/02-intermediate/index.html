Semantic HTML üìñ

Semantic HTML uses meaningful elements that describe the content's purpose, not just its appearance. This improves accessibility, SEO, and code maintainability.

Semantic Page Structure:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Semantic HTML Example</title>
</head>
<body>
    <header>
        <nav>
            <ul>
                <li><a href="#home">Home</a></li>
                <li><a href="#about">About</a></li>
                <li><a href="#contact">Contact</a></li>
            </ul>
        </nav>
    </header>

    <main>
        <article>
            <header>
                <h1>Article Title</h1>
                <p>Published on <time datetime="2024-03-15">March 15, 2024</time></p>
            </header>
            
            <section>
                <h2>Introduction</h2>
                <p>Article introduction content...</p>
            </section>
            
            <section>
                <h2>Main Content</h2>
                <p>Main article content...</p>
                
                <aside>
                    <p>Related information or sidebar content</p>
                </aside>
            </section>
        </article>
    </main>

    <footer>
        <p>© 2024 Your Website. All rights reserved.</p>
    </footer>
</body>
</html>

<header>

Introductory content or navigation

<nav>

Navigation links

<main>

Main content of the page

<article>

Independent, self-contained content

<section>

Thematic grouping of content

<aside>

Sidebar or related content

<footer>

Footer information

<time>

Date/time information

More Semantic Elements:

<!-- Address information -->
<address>
    Contact: <a href="mailto:john@example.com">john@example.com</a><br>
    Address: 123 Main St, City, State 12345
</address>

<!-- Figure with caption -->
<figure>
    <img src="chart.png" alt="Sales data chart">
    <figcaption>Fig 1. Sales performance for Q1 2024</figcaption>
</figure>

<!-- Details and summary -->
<details>
    <summary>Click to expand FAQ</summary>
    <p>Here's the detailed answer to the frequently asked question.</p>
</details>

<!-- Mark highlighted text -->
<p>Search results for <mark>HTML semantics</mark></p>

<!-- Progress indicator -->
<label for="progress">Loading:</label>
<progress id="progress" value="75" max="100">75%</progress>

Advanced Forms üìã

HTML5 introduced many new input types and form features that provide better user experience and built-in validation.

HTML5 Input Types:

<form action="/submit" method="post">
    <fieldset>
        <legend>Personal Information</legend>
        
        <!-- Basic inputs with validation -->
        <label for="email">Email:</label>
        <input type="email" id="email" name="email" required 
               placeholder="user@example.com">
        
        <label for="phone">Phone:</label>
        <input type="tel" id="phone" name="phone" 
               pattern="[0-9]{3}-[0-9]{3}-[0-9]{4}" 
               placeholder="123-456-7890">
        
        <label for="website">Website:</label>
        <input type="url" id="website" name="website" 
               placeholder="https://example.com">
        
        <label for="age">Age:</label>
        <input type="number" id="age" name="age" 
               min="18" max="100" step="1">
        
        <label for="salary">Salary Range:</label>
        <input type="range" id="salary" name="salary" 
               min="30000" max="200000" value="60000" 
               oninput="salaryOutput.value = this.value">
        <output id="salaryOutput">60000</output>
        
        <label for="birthday">Birthday:</label>
        <input type="date" id="birthday" name="birthday">
        
        <label for="appointment">Appointment Time:</label>
        <input type="datetime-local" id="appointment" name="appointment">
        
        <label for="color">Favorite Color:</label>
        <input type="color" id="color" name="color" value="#ff0000">
    </fieldset>
    
    <fieldset>
        <legend>File Upload</legend>
        
        <label for="resume">Upload Resume (PDF only):</label>
        <input type="file" id="resume" name="resume" 
               accept=".pdf,application/pdf" required>
        
        <label for="photos">Upload Photos (Multiple):</label>
        <input type="file" id="photos" name="photos" 
               accept="image/*" multiple>
    </fieldset>
    
    <button type="submit">Submit Form</button>
</form>

Advanced Form Validation:

<form novalidate>
    <!-- Custom validation patterns -->
    <label for="username">Username (alphanumeric, 3-15 chars):</label>
    <input type="text" id="username" name="username" 
           pattern="[a-zA-Z0-9]{3,15}" 
           title="Username must be 3-15 alphanumeric characters"
           required>
    
    <!-- Password with requirements -->
    <label for="password">Password (min 8 chars, 1 uppercase, 1 number):</label>
    <input type="password" id="password" name="password" 
           pattern="(?=.*[A-Z])(?=.*[0-9]).{8,}" 
           title="Password must be at least 8 characters with 1 uppercase and 1 number"
           required>
    
    <!-- Confirm password -->
    <label for="confirm-password">Confirm Password:</label>
    <input type="password" id="confirm-password" name="confirm-password" 
           required>
    
    <!-- Credit card with Luhn algorithm -->
    <label for="credit-card">Credit Card:</label>
    <input type="text" id="credit-card" name="credit-card" 
           pattern="[0-9]{13,19}" 
           placeholder="1234 5678 9012 3456"
           autocomplete="cc-number">
    
    <!-- Postal code by country -->
    <label for="postal-code">Postal Code:</label>
    <input type="text" id="postal-code" name="postal-code" 
           pattern="[0-9]{5}(-[0-9]{4})?" 
           placeholder="12345 or 12345-6789"
           autocomplete="postal-code">
    
    <button type="submit">Validate & Submit</button>
</form>

<form>

Media Elements üéµ

HTML5 provides native support for audio and video content without requiring plugins.

Video Element:

<!-- Basic video -->
<video controls width="640" height="360">
    <source src="video.mp4" type="video/mp4">
    <source src="video.webm" type="video/webm">
    <source src="video.ogg" type="video/ogg">
    <p>Your browser doesn't support HTML5 video. 
       <a href="video.mp4">Download the video</a> instead.</p>
</video>

<!-- Video with advanced features -->
<video controls autoplay muted loop poster="thumbnail.jpg" 
       preload="metadata" width="100%">
    <source src="video-hd.mp4" type="video/mp4">
    <source src="video-hd.webm" type="video/webm">
    
    <!-- Captions and subtitles -->
    <track kind="captions" src="captions-en.vtt" srclang="en" label="English">
    <track kind="subtitles" src="subtitles-es.vtt" srclang="es" label="Spanish">
    <track kind="descriptions" src="descriptions.vtt" srclang="en" label="Audio Descriptions">
    
    <p>Your browser doesn't support the video tag.</p>
</video>

Audio Element:

<!-- Basic audio -->
<audio controls>
    <source src="audio.mp3" type="audio/mpeg">
    <source src="audio.ogg" type="audio/ogg">
    <source src="audio.wav" type="audio/wav">
    <p>Your browser doesn't support the audio element.</p>
</audio>

<!-- Audio with advanced features -->
<audio controls preload="none" loop>
    <source src="podcast.mp3" type="audio/mpeg">
    <source src="podcast.ogg" type="audio/ogg">
    
    <!-- Fallback for older browsers -->
    <p>Your browser doesn't support HTML5 audio. 
       <a href="podcast.mp3">Download the audio file</a>.</p>
</audio>

Responsive Images with Picture Element:

<!-- Responsive images for different screen sizes -->
<picture>
    <source media="(min-width: 800px)" srcset="large-image.jpg">
    <source media="(min-width: 400px)" srcset="medium-image.jpg">
    <img src="small-image.jpg" alt="Responsive image description">
</picture>

<!-- Different image formats -->
<picture>
    <source srcset="image.avif" type="image/avif">
    <source srcset="image.webp" type="image/webp">
    <source srcset="image.jpg" type="image/jpeg">
    <img src="image.jpg" alt="Modern image formats">
</picture>

<!-- High DPI displays -->
<picture>
    <source srcset="image-2x.jpg 2x, image-1x.jpg 1x">
    <img src="image-1x.jpg" alt="High DPI image">
</picture>

controls

Shows playback controls

autoplay

Starts playing automatically (use with muted)

muted

Starts muted (required for autoplay)

loop

Repeats the media

preload

none, metadata, or auto

poster

Thumbnail image for video

üì± Media Best Practices:

  • Always provide multiple format sources for compatibility
  • Include fallback content for unsupported browsers
  • Use appropriate preload settings to optimize performance
  • Add captions and descriptions for accessibility
  • Consider autoplay policies (usually requires muted)
  • Optimize file sizes for web delivery

Accessibility (A11y) ‚ôø

Web accessibility ensures that websites are usable by people with disabilities. Proper HTML structure and ARIA attributes are essential.

ARIA Attributes:

<!-- Landmark roles -->
<header role="banner">
    <nav role="navigation" aria-label="Main navigation">
        <ul>
            <li><a href="/" aria-current="page">Home</a></li>
            <li><a href="/about">About</a></li>
            <li><a href="/contact">Contact</a></li>
        </ul>
    </nav>
</header>

<main role="main">
    <article>
        <h1>Article Title</h1>
        <p>Article content...</p>
    </article>
</main>

<aside role="complementary" aria-label="Related articles">
    <h2>Related Content</h2>
    <ul>
        <li><a href="/related1">Related Article 1</a></li>
        <li><a href="/related2">Related Article 2</a></li>
    </ul>
</aside>

<footer role="contentinfo">
    <p>© 2024 Company Name</p>
</footer>

Accessible Forms:

<form>
    <fieldset>
        <legend>Personal Information</legend>
        
        <!-- Explicit labels -->
        <label for="full-name">Full Name (required):</label>
        <input type="text" id="full-name" name="full-name" 
               required aria-describedby="name-help">
        <div id="name-help" class="help-text">
            Enter your first and last name
        </div>
        
        <!-- Error handling -->
        <label for="email-address">Email Address:</label>
        <input type="email" id="email-address" name="email" 
               aria-describedby="email-error" aria-invalid="true">
        <div id="email-error" role="alert" class="error-message">
            Please enter a valid email address
        </div>
        
        <!-- Radio button group -->
        <fieldset>
            <legend>Preferred Contact Method</legend>
            <input type="radio" id="contact-email" name="contact-method" value="email">
            <label for="contact-email">Email</label>
            
            <input type="radio" id="contact-phone" name="contact-method" value="phone">
            <label for="contact-phone">Phone</label>
        </fieldset>
    </fieldset>
    
    <button type="submit" aria-describedby="submit-help">
        Submit Form
    </button>
    <div id="submit-help" class="help-text">
        Review your information before submitting
    </div>
</form>

Interactive Elements:

<!-- Custom button -->
<div role="button" tabindex="0" 
     aria-pressed="false" 
     onclick="toggleButton(this)" 
     onkeydown="handleKeyDown(event, this)">
    Custom Toggle Button
</div>

<!-- Expandable content -->
<button aria-expanded="false" 
        aria-controls="expandable-content" 
        onclick="toggleExpanded(this)">
    Show More Information
</button>
<div id="expandable-content" hidden>
    <p>Additional content that can be shown or hidden.</p>
</div>

<!-- Progress indicator -->
<div role="progressbar" 
     aria-valuenow="75" 
     aria-valuemin="0" 
     aria-valuemax="100" 
     aria-label="Upload progress">
    <div class="progress-bar" style="width: 75%"></div>
</div>

<!-- Live region for dynamic updates -->
<div aria-live="polite" aria-atomic="true" id="status-message">
    <!-- Status updates will be announced by screen readers -->
</div>

‚ôø Accessibility Checklist:

  • Use semantic HTML elements correctly
  • Provide alternative text for images
  • Ensure proper color contrast (4.5:1 minimum)
  • Make all interactive elements keyboard accessible
  • Use ARIA attributes when HTML semantics aren't enough
  • Test with screen readers and keyboard navigation
  • Provide skip links for navigation
  • Use proper heading hierarchy

Meta Tags & SEO üîç

Meta tags provide metadata about your HTML document. They're crucial for SEO, social media sharing, and browser behavior.

Essential Meta Tags:

<head>
    <!-- Character encoding (must be first) -->
    <meta charset="UTF-8">
    
    <!-- Viewport for responsive design -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    
    <!-- Page title (appears in browser tab and search results) -->
    <title>Page Title | Website Name</title>
    
    <!-- Meta description (appears in search results) -->
    <meta name="description" 
          content="A brief, compelling description of your page content (150-160 characters)">
    
    <!-- Keywords (less important for modern SEO) -->
    <meta name="keywords" content="html, css, javascript, web development, tutorial">
    
    <!-- Author information -->
    <meta name="author" content="Your Name">
    
    <!-- Robots instructions -->
    <meta name="robots" content="index, follow">
    
    <!-- Canonical URL (prevents duplicate content issues) -->
    <link rel="canonical" href="https://www.example.com/page">
    
    <!-- Language -->
    <meta name="language" content="English">
    <link rel="alternate" hreflang="es" href="https://www.example.com/es/page">
</head>

<!-- Favicon links -->

<link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png">

üéØ SEO Best Practices:

  • Write unique, descriptive titles (50-60 characters)
  • Craft compelling meta descriptions (150-160 characters)
  • Use proper heading hierarchy (H1 ‚Üí H2 ‚Üí H3)
  • Add alt text to all images
  • Use semantic HTML elements
  • Implement structured data (JSON-LD)
  • Ensure fast loading times
  • Make your site mobile-friendly
