What is HTML? üåê

HTML (HyperText Markup Language) is the standard markup language for creating web pages. It describes the structure and content of a webpage using elements and tags.

Key Points:

  • HTML uses tags to define elements
  • Tags are enclosed in angle brackets: <tagname>
  • Most tags come in pairs: opening and closing tags
  • HTML provides structure, not styling (that's CSS)

Basic HTML Structure:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My First Webpage</title>
</head>
<body>
    <h1>Hello, World!</h1>
    <p>This is my first HTML page.</p>
</body>
</html>

Document Structure üèóÔ∏è

Every HTML document follows a specific structure with essential elements.

Complete Document Structure:

<!DOCTYPE html>           <!-- Document type declaration -->
<html lang="en">              <!-- Root element -->
<head>                        <!-- Document metadata -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta name="description" content="Page description">
    <title>Page Title</title>
    <link rel="stylesheet" href="styles.css">
</head>
<body>                        <!-- Visible content -->
    <h1>Main Heading</h1>
    <p>Page content goes here</p>
</body>
</html>

<!DOCTYPE html>

Tells the browser this is an HTML5 document

<html>

Root element that contains all other elements

<head>

Contains metadata not visible to users

<body>

Contains all visible content

Headings & Paragraphs üìÑ

Headings create hierarchy and structure. Paragraphs contain blocks of text.

Heading Hierarchy:

<h1>Main Title (Most Important)</h1>
<h2>Section Title</h2>
<h3>Subsection Title</h3>
<h4>Sub-subsection Title</h4>
<h5>Minor Heading</h5>
<h6>Least Important Heading</h6>

<p>This is a paragraph. It can contain multiple sentences 
and will wrap to the next line automatically when it reaches 
the edge of its container.</p>

<p>Each paragraph element creates a new block of text 
with spacing above and below it.</p>

Links üîó

Links connect web pages and resources together. They're created with the anchor tag <a>.

Basic Links:

<!-- External link -->
<a href="https://www.example.com">Visit Example.com</a>

<!-- Internal link (same site) -->
<a href="about.html">About Us</a>

<!-- Link to section on same page -->
<a href="#section1">Go to Section 1</a>

<!-- Email link -->
<a href="mailto:someone@example.com">Send Email</a>

<!-- Phone link -->
<a href="tel:+1234567890">Call Us</a>

Link Attributes:

<!-- Open in new tab -->
<a href="https://example.com" target="_blank">Open in New Tab</a>

<!-- Download link -->
<a href="document.pdf" download>Download PDF</a>

<!-- Link with title (tooltip) -->
<a href="https://example.com" title="Visit Example Website">Example</a>

<!-- Relationship attributes -->
<a href="https://example.com" rel="noopener noreferrer" target="_blank">
    Secure External Link
</a>

Images üñºÔ∏è

Images make web pages visually appealing and informative. The <img> tag embeds images.

Basic Image Syntax:

<!-- Basic image -->
<img src="image.jpg" alt="Description of image">

<!-- Image with dimensions -->
<img src="photo.png" alt="A beautiful landscape" width="300" height="200">

<!-- Responsive image -->
<img src="responsive.jpg" alt="Responsive image" style="max-width: 100%; height: auto;">

Image with Different Sources:

<!-- Image with fallback -->
<img src="image.webp" alt="Modern image format" 
     onerror="this.src='fallback.jpg'">

<!-- Using figure element -->
<figure>
    <img src="chart.png" alt="Sales chart for 2024">
    <figcaption>Sales performance chart for 2024</figcaption>
</figure>

src attribute

Specifies the path to the image file

alt attribute

Provides alternative text for accessibility and SEO

width/height

Sets image dimensions (use CSS instead for responsive design)

title attribute

Shows tooltip text when hovering over image

üåü Best Practices:

  • Always include meaningful alt text
  • Use appropriate image formats (WebP, JPEG, PNG)
  • Optimize images for web (compress file sizes)
  • Use responsive images for different screen sizes
  • Consider lazy loading for performance

Tables display data in rows and columns. They should only be used for tabular data, not layout.

Basic Table Structure:

<table>
    <thead>
        <tr>
            <th>Name</th>
            <th>Age</th>
            <th>City</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>John Doe</td>
            <td>25</td>
            <td>New York</td>
        </tr>
        <tr>
            <td>Jane Smith</td>
            <td>30</td>
            <td>London</td>
        </tr>
    </tbody>
</table>

Basic Forms üìù

Forms collect user input and send it to a server for processing.

Simple Contact Form:

<form action="/submit" method="post">
    <label for="name">Name:</label>
    <input type="text" id="name" name="name" required>
    
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required>
    
    <label for="message">Message:</label>
    <textarea id="message" name="message" rows="4"></textarea>
    
    <button type="submit">Send Message</button>
</form>

Common Input Types:

<!-- Text inputs -->
<input type="text" placeholder="Enter text">
<input type="email" placeholder="email@example.com">
<input type="password" placeholder="Password">
<input type="number" min="1" max="100">
<input type="tel" placeholder="Phone number">
<input type="url" placeholder="https://example.com">

<!-- Date and time -->
<input type="date">
<input type="time">
<input type="datetime-local">

<!-- Choices -->
<input type="checkbox" id="agree">
<label for="agree">I agree to terms</label>

<input type="radio" id="male" name="gender" value="male">
<label for="male">Male</label>
<input type="radio" id="female" name="gender" value="female">
<label for="female">Female</label>

<!-- Selection -->
<select name="country">
    <option value="us">United States</option>
    <option value="uk">United Kingdom</option>
    <option value="ca">Canada</option>
</select>

üìã Form Best Practices:

  • Always use labels with form controls
  • Use appropriate input types for validation
  • Add required attributes for mandatory fields
  • Use placeholder text for hints
  • Group related fields with fieldset

Attributes ‚ö°

Attributes provide additional information about HTML elements and modify their behavior.

Common Global Attributes:

<!-- ID: Unique identifier -->
<div id="header">Header content</div>

<!-- Class: For styling and selection -->
<p class="highlight important">Important text</p>

<!-- Title: Tooltip text -->
<span title="This is a tooltip">Hover over me</span>

<!-- Style: Inline CSS -->
<p style="color: red; font-weight: bold;">Red bold text</p>

<!-- Data attributes: Custom data -->
<div data-user-id="123" data-role="admin">User info</div>

<!-- Language -->
<p lang="es">Hola mundo</p>

<!-- Hidden -->
<div hidden>This content is hidden</div>

Element-Specific Attributes:

<!-- Link attributes -->
<a href="https://example.com" target="_blank" rel="noopener">Link</a>

<!-- Image attributes -->
<img src="photo.jpg" alt="Description" width="300" height="200">

<!-- Input attributes -->
<input type="text" name="username" placeholder="Enter username" 
       required maxlength="20" pattern="[a-zA-Z0-9]+">

<!-- Form attributes -->
<form action="/submit" method="post" enctype="multipart/form-data">

<!-- Table attributes -->
<td colspan="2" rowspan="3">Cell content</td>

id

Unique identifier for the element

class

One or more class names for styling

style

Inline CSS styles

title

Tooltip text on hover

data-*

Custom attributes for JavaScript

hidden

Hides the element
